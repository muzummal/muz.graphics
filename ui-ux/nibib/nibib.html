NIBIB Website Redesign and Applying USWDS

A redesign project to update NIBIB's website with the U.S. Web Design System.

Overview

Summary

The purpose of this project was to improve my knowledge, skills, and abilities on the U.S. Web Design System (USWDS).

The National Institute of Biomedical Imaging and Bioengineering’s (NIBIB) webpage was launched approximately three to four years prior to this project. Although they had implemented some elements from the USWDS, there were a few areas which needed improvement. More specifically, the page’s hierarchy, and better integration sections.

Understand

Research

I conducted in-depth research of the USWDS’s website to learn and understand their goals, processes, and designs. I reviewed multiple government websites and compared how each used the system. It was interesting to learn that organizations do not use all of the USWDS.

Competitive Audit

Reviewing government websites implementing the USWDS increased my learning and understanding of how to incorporate the system in web design. I observed how some organizations fully implemented the system, while others only used some elements based on the purpose for the site, the user demographic, and other custom needs. It was these observations that inspired my project to redesign the NIBIB website.

Product Redesign

Wireframes

Hand drawn wireframes are still the best way to outline or map and develop a website design. I first chose to create a hard copy outline for my base USWDS wireframe. Using this wireframe allowed me to layout a design and mix and match ideas in many iterations throughout the process. I even referred to my manual drawings when creating the digital wireframes.

Digital Wireframes

Accessibility

The USWDS’s mission is to shape the future of government digital services. This system makes it easier to implement design decisions that improve user navigation experience. As previously mentioned, NIBIB needed improvement in some design elements.

With the hierarchy it needed clarification and organization. For example, regarding clarification, page headers needed size adjustments. With respect to the organization, page sections needed to be moved to a call out box or separation from the main content. As for the integrated sections on the homepage, they needed a better color language and layout.

Conclusion

Take Away

The scope of this project focused on improving the implementation of USWDS elements on the NIBIB website. Many of my design choices were guided by both the USWDS and general web design standards. I kept in mind that there might be a specific reason for the order and layout of some items on the site. Therefore, I chose to keep elements as close as possible to their original location.

My takeaway is that a design system is very helpful for many reasons such as providing specific design rules and appropriate tools. Additionally, the system helps keep designers focused and organized in creating accessible websites. Big organizational branches such as NIBIB sometimes struggle to keep their websites clean and organized. Some reasons could be inadequate allocation of resources, lack of UX Designers on the team, and high web content volume. You can see the progression through the paper and digital wireframes as I incorporate USWDS into NIBIB’s website. My creativity allowed me to use USWDS as a tool to improve on what I know about user interface and user experience. I created an eye-catching functional design with a unique flair.
